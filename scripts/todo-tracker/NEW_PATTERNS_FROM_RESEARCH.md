# New Patterns from Developer Research

**Date:** 10-11-2025  
**Source:** Internet research on AI-generated code patterns reported by developers

---

## 1. False Assurance Patterns

### Patterns That Look Complete But Aren't

```javascript
// "seems to work" - Code appears functional but may have hidden issues
{ regex: /seems.*to.*work|appears.*to.*work|looks.*like.*it.*works/gi, type: "FALSE_ASSURANCE", severity: "HIGH", category: "deceptive" },

// "looks correct" - Deceptive confidence
{ regex: /looks.*correct|appears.*correct|seems.*correct/gi, type: "FALSE_ASSURANCE", severity: "HIGH", category: "deceptive" },

// "appears complete" - But isn't actually complete
{ regex: /appears.*complete|seems.*complete|looks.*complete/gi, type: "FALSE_ASSURANCE", severity: "CRITICAL", category: "deceptive" },

// "probably works" - Uncertainty disguised as confidence
{ regex: /probably.*works|maybe.*works|possibly.*works|could.*work/gi, type: "FALSE_ASSURANCE", severity: "HIGH", category: "deceptive" },
```

## 2. Uncertain Language Patterns

### AI-Generated Uncertainty Markers

```javascript
// Tentative language indicating lack of confidence
{ regex: /\b(maybe|probably|possibly|could be|might be|seems like|appears to be)\b/gi, type: "UNCERTAIN_LANGUAGE", severity: "MEDIUM", category: "deceptive" },

// "As per the requirements" - AI-generated phrasing
{ regex: /as per.*requirements|in order to achieve|according to.*spec/gi, type: "AI_GENERATED_PHRASING", severity: "LOW", category: "deceptive" },
```

## 3. Overly Generic Comments

### AI-Generated Redundant Comments

```javascript
// "This function does X" - Overly descriptive without insight
{ regex: /this function.*does|this function.*calculates|this function.*handles|this function.*processes/gi, type: "OVERLY_GENERIC_COMMENT", severity: "LOW", category: "deceptive" },

// "Initialize variable" - Stating the obvious
{ regex: /initialize.*variable|set.*variable.*to|create.*variable/gi, type: "REDUNDANT_COMMENT", severity: "LOW", category: "deceptive" },

// Generic "handles X" without details
{ regex: /handles.*(?:user|data|input|request)/gi, type: "VAGUE_COMMENT", severity: "MEDIUM", category: "deceptive" },
```

---

## 4. Deceptive Comment Patterns

### Comments That Mislead

```javascript
// "Fix later" - Without context
{ regex: /fix.*later|fix.*someday|fix.*eventually|address.*later/gi, type: "DEFERRED_FIX", severity: "MEDIUM", category: "temporal" },

// "TODO: Implement this function" - Generic placeholder
{ regex: /TODO.*implement.*(?:this|that|function|feature)/gi, type: "GENERIC_TODO", severity: "HIGH", category: "incomplete" },

// "This function handles X" without security considerations
{ regex: /this function handles.*(?:auth|login|password|session)/gi, type: "INSECURE_COMMENT", severity: "CRITICAL", category: "deceptive" },
```

## 6. Security-Related Patterns

### Hardcoded Secrets (AI Often Includes These)

```javascript
// Hardcoded credentials
{ regex: /\b(API_KEY|SECRET_KEY|PASSWORD|TOKEN|ACCESS_KEY|api_key|secret_key|password|token|access_key)\s*=\s*['"][^'"]+['"]/gi, type: "HARDCODED_SECRET", severity: "CRITICAL", category: "security" },

// Hardcoded passwords/tokens in comments
{ regex: /password.*[:=]\s*['"]|token.*[:=]\s*['"]|secret.*[:=]\s*['"]/gi, type: "HARDCODED_SECRET", severity: "CRITICAL", category: "security" },
```

### Insecure Input Handling

```javascript
// Dangerous functions without validation
{ regex: /\b(eval|exec|input|raw_input|query|unescape)\s*\(/gi, type: "INSECURE_INPUT", severity: "CRITICAL", category: "security" },
```

## 7. Incomplete Error Handling

### Missing Error Handling Patterns

```javascript
// Code that should have error handling but doesn't
{ regex: /(?:fetch|axios|request|query|execute).*(?!try|catch|\.catch)/gi, type: "MISSING_ERROR_HANDLING", severity: "HIGH", category: "incomplete" },
```

## 8. Deceptive Success Patterns

### False Success Indicators

```javascript
// "Completed successfully" but with caveats
{ regex: /completed successfully.*(?:but|however|actually|really|though)/gi, type: "FALSE_SUCCESS", severity: "HIGH", category: "deceptive" },

// "Works" but with limitations
{ regex: /works.*(?:but|however|except|unless|if)/gi, type: "CONDITIONAL_SUCCESS", severity: "HIGH", category: "deceptive" },
```

## 9. Pattern Categories Summary

### New Categories to Add:

1. **FALSE_ASSURANCE** - Code that appears complete but isn't
2. **UNCERTAIN_LANGUAGE** - Tentative language indicating uncertainty
3. **OVERLY_GENERIC_COMMENT** - Redundant AI-generated comments
4. **DEFERRED_FIX** - "Fix later" without context
5. **AI_GENERATED_MARKER** - Explicit AI generation indicators
6. **HARDCODED_SECRET** - Security vulnerability
7. **INSECURE_INPUT** - Dangerous input handling
8. **MISSING_ERROR_HANDLING** - Incomplete error handling
9. **CONDITIONAL_SUCCESS** - Success with caveats

## 10. Priority Recommendations

### Critical (Must Fix):
- `FALSE_ASSURANCE` (appears complete)
- `HARDCODED_SECRET`
- `INSECURE_INPUT`
- `INSECURE_COMMENT` (auth without security)

### High (Should Fix):
- `FALSE_ASSURANCE` (probably works)
- `DEFERRED_FIX`
- `MISSING_ERROR_HANDLING`
- `CONDITIONAL_SUCCESS`

### Medium (Review):
- `UNCERTAIN_LANGUAGE`
- `VAGUE_COMMENT`
- `GENERIC_TODO`

### Low (Informational):
- `OVERLY_GENERIC_COMMENT`
- `REDUNDANT_COMMENT`
- `AI_GENERATED_MARKER`
- `AI_GENERATED_PHRASING`

## Implementation Notes

1. **False Positives:** Some patterns may have legitimate uses (e.g., "maybe" in test code)
2. **Context Matters:** Consider surrounding code context
3. **Security Patterns:** These should always be flagged as CRITICAL
4. **AI Markers:** Low severity but useful for tracking AI-generated code
