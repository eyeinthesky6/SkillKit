## Maintainer Responsibilities

As a maintainer of this project, you are expected to:

1. **Code Quality**
   - Review and merge pull requests
   - Ensure code follows project standards
   - Maintain test coverage

2. **Community**
   - Triage issues and pull requests
   - Help onboard new contributors
   - Participate in discussions and decision making

3. **Releases**
   - Help with versioning and changelog management
   - Publish releases
   - Update documentation

4. **Security**
   - Respond to security reports
   - Keep dependencies updated
   - Follow security best practices

## Becoming a Maintainer

We welcome new maintainers who are interested in contributing to the project. To become a maintainer:

1. Have at least 3 significant pull requests merged
2. Demonstrate understanding of the project's goals and architecture
3. Be active in the community (issues, discussions, etc.)
4. Be nominated by an existing maintainer
5. Be approved by a majority of current maintainers
