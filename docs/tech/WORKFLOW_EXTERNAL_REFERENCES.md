## Recommendations

### For Workflow Templates

1. **Make external references optional:**
   - Use conditional language: "If `docs/AITracking/` exists..."
   - Use `2>/dev/null` in commands (already done)
   - Make references in comments/checklists, not required steps

2. **Document expected external docs:**
   - List which external docs are optional vs required
   - Provide guidance on creating missing docs

3. **Consider making references conditional:**
   - Check if file exists before referencing
   - Provide alternative instructions if missing

### For Projects

**Optional but recommended:**
- `docs/AITracking/` - For tracking AI-assisted work
- `CHANGELOG.md` - Standard project documentation
- `AGENTS.md` - Generated by SkillKit (`tsk sync`)

**Project-specific (may not exist):**
- `docs/design-system.md` - Only if project has one
- `docs/api-changelog.md` - Only if project tracks API changes
- `docs/SKILLKIT_CUSTOMIZATIONS.md` - Only if customizations made
