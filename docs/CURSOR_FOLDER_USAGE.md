# .cursor Folder Usage Policy

## Overview

The `.cursor/` folder is **IDE-specific** and should **ONLY** contain:
- **Commands** (`.cursor/commands/*.md`) - Workflow files for Cursor IDE
- **Rules** (`.cursor/rules/*.mdc`) - IDE rules and guidelines

## ❌ What Should NEVER Be in .cursor/

- **Reports** - All reports go to `docs/` subdirectories
- **Logs** - Use `logs/` or `docs/skillkit/`
- **Cache files** - Use `.cache/` or project-specific cache directories
- **Generated data** - Use `docs/` or project-specific output directories

## ✅ Correct Report Locations

### SkillKit Reports: `docs/skillkit/`
- Diagnostics: `docs/skillkit/diagnostics-{timestamp}.{md|json}`
- All SkillKit-generated diagnostic reports

### Audit Reports: `docs/audit/`
- System health checks: `docs/audit/audit-report-{date}.{md|json}`

### TODO Tracker: `docs/todo-tracker/`
- TODO analysis: `docs/todo-tracker/Comprehensive_TODO_Analysis_{timestamp}.{md|json|html}`

## ✅ .cursor/ Folder Contents

### `.cursor/commands/` - Workflow Files
- **Purpose:** Cursor IDE reads workflow markdown files from here
- **Generated by:** `tsk workflow` or `tsk init --cursor`
- **Format:** Markdown files (`.md`)
- **Project-specific:** ✅ YES - Adapted based on detected languages/tools
- **Examples:**
  - `BEGIN_SESSION.md` - Adapted with project-specific commands
  - `IMPLEMENT_FEATURE.md` - Contains language-specific setup sections
  - `FIX_BUGS.md` - Uses project's lint/test commands

### `.cursor/rules/` - IDE Rules
- **Purpose:** Cursor IDE rules and guidelines
- **Generated by:** `tsk init --rules`
- **Format:** Markdown files (`.mdc`)
- **Project-specific:** ⚠️ Currently generic - could be enhanced
- **Examples:**
  - `skillkit-workflows.mdc` - Workflow routing rules

## Project-Specific Generation

### Commands (`.cursor/commands/*.md`)

**Status:** ✅ **PROJECT-SPECIFIC**

Workflow files are adapted based on:
1. **Detected languages** (Python, TypeScript, JavaScript, etc.)
2. **Package managers** (poetry, pnpm, npm, yarn, etc.)
3. **Tools** (ruff, eslint, pytest, jest, etc.)
4. **Project scripts** (from `package.json` or `pyproject.toml`)

**Example:**
```markdown
# Generated for Python + TypeScript monorepo

## Phase 2.1: Language-Specific Setup

### PYTHON (.)
```bash
cd .
poetry install
cd -
```

### TYPESCRIPT (apps/web)
```bash
cd apps/web
pnpm install
cd -
```
```

### Rules (`.cursor/rules/*.mdc`)

**Status:** ⚠️ **GENERIC** (Could be enhanced)

Current rules are generic workflow routing. Could be enhanced to include:
- Project-specific command patterns
- Language-specific guidelines
- Framework-specific best practices

## Verification

```bash
# Check for any reports in .cursor/
find .cursor -name "*.json" -o -name "*report*" -o -name "*.log"

# Should return nothing (or only .md/.mdc files)
```
